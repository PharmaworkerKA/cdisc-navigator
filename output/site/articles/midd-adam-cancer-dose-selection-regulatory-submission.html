MIDD×ADaM：癌臨床試験の用量選択と規制提出最適化

Photo from Unsplash

この記事でわかること

  • MIDD（Model-Informed Drug Development）の概念と、癌臨床試験におけるADaM（Analysis Data Model）の重要な役割を理解できます。
  • 用量選択を効果的にサポートするためのADaM設計パターンと具体的なSASコード例を学べます。
  • FDA（米国食品医薬品局）およびPMDA（医薬品医療機器総合機構）への規制提出における、ADaM活用の最新トレンドと注意点がわかります。
  • ADaMデータセット（特にBDS、ADTTE）の具体的な作り方と、派生変数ロジックの実装方法を習得できます。

結論から言うと

癌臨床試験における最適な用量選択と規制提出には、MIDDの導入が不可欠であり、その実現にはADaMデータの適切な設計と活用が鍵となります。特に、曝露反応関係を評価するためのADaM BDS（Basic Data Structure）やADTTE（Analysis Data for Time-to-Event）の作成は、承認申請の成功を左右する重要な要素です。海外の最新動向を踏まえ、日本の実務に合わせたADaM実装を進めましょう。

MIDD（Model-Informed Drug Development）とは？

MIDD（Model-Informed Drug Development、モデルに基づく医薬品開発）とは、薬物動態（PK）、薬力学（PD）、臨床アウトカム、患者特性などのデータを統合し、数理モデルを用いて医薬品開発の意思決定を支援するアプローチです。これにより、臨床試験のデザイン最適化、用量選択の合理化、希少疾患でのエビデンス補完など、開発の効率と成功率を向上させることができます。

癌臨床試験において、MIDDは特にその重要性を増しています。新規抗癌剤は、しばしば限られた患者数での開発を強いられ、用量設定や有効性の評価が難しい場合があります。MIDDを活用することで、限られた臨床データからより多くの情報を引き出し、最適な用量やレジメンを推定し、より迅速かつ効果的な開発が可能になります。FDAもMIDDの活用を強く推奨しており、承認申請における重要な要素として注目されています。

実務での使い方・実装手順：ADaM設計と用量選択支援

MIDD解析を効果的に行うためには、ADaM（Analysis Data Model）の適切な設計が不可欠です。特に、用量選択や曝露反応解析を目的としたADaMデータセットでは、PKデータ、曝露量、用量情報、および関連する臨床エンドポイントを効率的に統合し、解析しやすい形に加工する必要があります。ここでは、その具体的なADaM設計パターンとSASコード例をご紹介します。

1. ADSL（Subject-Level Analysis Dataset）への曝露量情報の集約

ADSLは被験者レベルの解析データセットであり、各被験者の重要な背景情報、治療情報、そして曝露情報を集約します。MIDD解析では、各被験者の実際の曝露量（例：AUCやCmax）が非常に重要になります。

派生変数ロジック: * AVISITN (Analysis Visit Number): 各曝露量測定時点の数値表現。 * CUMDOSE (Cumulative Dose): 薬剤投与開始からの累積総投与量。SDTMのEX（Exposure）ドメインから計算します。 * DOSECATEG (Dose Category): 曝露量（例：AUC、Cmax、または平均定常状態濃度）に基づいて、事前に定義されたカテゴリー（例：低用量、中用量、高用量）に分類します。これは、用量反応関係を可視化する際によく用いられます。

SASコード例（ADSLへの累積用量と初回曝露量カテゴリの追加）:

/* SDTM EXドメインから累積用量を計算 */
data ex_cume;
    set sdtm.ex;
    by usubjid extrt;
    retain cume_dose 0;
    if first.usubjid then cume_dose = 0;
    if exdose is not missing then cume_dose = cume_dose + exdose;
    output;
run;

proc sort data=ex_cume out=ex_last(keep=usubjid extrt cume_dose);
    by usubjid extrt descending exstdtc descending exsttm;
run;
data ex_last;
    set ex_last;
    by usubjid extrt;
    if first.usubjid;
    rename cume_dose=CUMDOSE;
run;

/* ADSLへのマージ例 */
data adsl_temp;
    merge adsl(in=a)
          ex_last(in=b keep=usubjid CUMDOSE)
    ;
    by usubjid;

    /* 仮の曝露量データ（PKドメイン等から結合されることを想定）*/
    /* ここでは簡略化のためランダム値を生成しますが、実際にはPK解析結果から取得 */
    call ranuni(seed, exposure_val);
    if exposure_val <= 0.3 then DOSECATEG = 'Low Exposure';
    else if exposure_val <= 0.7 then DOSECATEG = 'Medium Exposure';
    else DOSECATEG = 'High Exposure';

    length DOSECATEG $20.;
run;

/* ADSLデータセットの最終化 */
data adsl;
    set adsl_temp(drop=exposure_val);
run;

2. ADaM BDS (Basic Data Structure) for PK/PD and Exposure-Response

MIDD解析の主役となるのがBDSです。特に、曝露反応関係を評価するためのADaM BDSは、PKパラメータ、薬力学的エンドポイント（例：腫瘍縮小率、バイオマーカー値）、および用量情報を統合します。

派生変数ロジック: * AVAL (Analysis Value): 解析の主要な結果値（例：AUC、Cmax、腫瘍縮小率、特定バイオマーカー値）。 * PARAM (Parameter): AVALが示す解析パラメータの名称（例：AUC(0-INF)、Tumor Response (%)）。 * BASE (Baseline Value): 薬物投与前のベースライン値。例えば、腫瘍サイズやバイオマーカーのベースライン値。 * CHG (Change from Baseline): ベースラインからの変化量。 * PCHG (Percent Change from Baseline): ベースラインからの変化率。 * EXPOSET (Exposure Set): PKまたは曝露関連の解析に使用されることを示すフラグ。'Y'または'N'。 * DOSEGRP (Dose Group): 被験者が属する用量群。これは、ADSLのDOSECATEGとは異なる場合があります。

SASコード例（曝露反応解析用BDSの作成）:

/* 仮のPKデータセット（実際はPK解析結果から）*/
data pk_data;
    input USUBJID $ AVISITN AUC CMAX;
    cards;
    CDISC-001-1001 1 1200 80
    CDISC-001-1001 2 1500 95
    CDISC-001-1002 1 1000 70
    CDISC-001-1003 1 1350 88
    CDISC-001-1003 2 1600 102
    ;
run;

/* 仮の腫瘍反応データセット（SDTM RSドメインなどから）*/
data tumor_response;
    input USUBJID $ AVISITN BASELINE_TUMOR_SIZE CURRENT_TUMOR_SIZE;
    cards;
    CDISC-001-1001 1 50 45
    CDISC-001-1001 2 50 30
    CDISC-001-1002 1 60 55
    CDISC-001-1003 1 45 40
    CDISC-001-1003 2 45 25
    ;
run;

/* PKデータからBDSを作成 */
data adpk_bds_auc;
    set pk_data;
    PARAMCD = 'AUC';
    PARAM = 'AUC(0-INF)';
    AVAL = AUC;
    EXPOSET = 'Y';
    output;
run;

data adpk_bds_cmax;
    set pk_data;
    PARAMCD = 'CMAX';
    PARAM = 'Cmax';
    AVAL = CMAX;
    EXPOSET = 'Y';
    output;
run;

/* 腫瘍反応データからBDSを作成 */
data adtumor_bds_response;
    set tumor_response;
    PARAMCD = 'TUMORSP';
    PARAM = 'Tumor Response (%)';
    BASE = BASELINE_TUMOR_SIZE;
    CHG = CURRENT_TUMOR_SIZE - BASELINE_TUMOR_SIZE;
    PCHG = (CHG / BASELINE_TUMOR_SIZE) * 100;
    AVAL = PCHG;
    EXPOSET = 'N';
    output;
run;

/* 最終的なADaM BDSの結合 */
data adbds;
    length PARAMCD PARAM $30. EXPOSET $1.;
    set adpk_bds_auc
        adpk_bds_cmax
        adtumor_bds_response;

    /* ADSLから必要な情報をマージ */
    merge adbds(in=a)
          adsl(in=b keep=USUBJID TRT01A DOSECATEG CUMDOSE)
    ;
    by USUBJID;

    /* AVALは数値、PARAMは文字列であることを明確に */
    format AVAL 8.2;
run;

3. ADTTE (Analysis Data for Time-to-Event) for Dose-Response Survival Analysis

癌臨床試験では、PFS（無増悪生存期間）やOS（全生存期間）のような時間イベントデータが中心となります。ADaM ADTTEは、これらの時間イベント解析を目的としたデータセットで、用量群間の生存期間の比較によく用いられます。

派生変数ロジック: * ADTF (Analysis Date/Time First): イベントの発生日または最終追跡日。 * ADTL (Analysis Date/Time Last): イベントの発生日または最終追跡日。 * CNSR (Censor Variable): イベントが発生した場合は0、打ち切りの場合は1。 * ATPT (Analysis Time Point): 薬物投与開始からのイベントまでの期間（日または月）。 * PARAMCD (Parameter Code): 解析パラメータコード（例：PFS、OS）。 * PARAM (Parameter): 解析パラメータの名称（例：Progression Free Survival (Months)）。

SASコード例（ADTTE作成の概念）:

/* 仮のSDTM AE (有害事象) ドメインからの死亡日 */
data sdtm.ae_death;
    input USUBJID $ AESDTC $;
    format AESDTC yymmdd10.;
    datalines;
    CDISC-001-1001 2023-01-15
    CDISC-001-1002 2022-12-01
    ;
run;

/* 仮のSDTM TU/TR (腫瘍評価) ドメインからのPD (病勢進行) 日 */
data sdtm.tu_pd;
    input USUBJID $ TUDTC $ TRTESTCD $ TREVAL $;
    format TUDTC yymmdd10.;
    datalines;
    CDISC-001-1001 2022-09-01 PROGRESSION INVESTIGATOR
    CDISC-001-1002 2022-08-15 PROGRESSION INVESTIGATOR
    CDISC-001-1003 2022-10-20 PROGRESSION INVESTIGATOR
    ;
run;

/* ADTTE (PFS) の作成例 */
data adtte_pfs;
    merge adsl(in=a)
          sdtm.tu_pd(in=b where=(TRTESTCD='TARGET' and TREVAL='PROGRESSION'))
    ;
    by USUBJID;

    /* 治療開始日 (TRT01SDTC) はADSLから取得 */
    /* PD日がない場合は最終観察日 (LSTOBSDT) を使用 */
    if b then ADTF = input(TUDTC,yymmdd10.); /* PD日 */
    else ADTF = input(LSTOBSDT,yymmdd10.); /* 最終観察日（架空）*/

    /* 治療開始日 (架空の変数) */
    TRTSTDTC = input(TRT01SDTC,yymmdd10.); 

    ATPT = intck('MONTH', TRTSTDTC, ADTF); /* 治療開始からイベントまでの期間 (月) */

    CNSR = 1; /* 初期値は打ち切り */
    if b then CNSR = 0; /* PDイベント発生 */

    PARAMCD = 'PFS';
    PARAM = 'Progression Free Survival (Months)';

    /* ADSLから用量カテゴリなどの情報をマージ */
    keep USUBJID TRT01A DOSECATEG ATPT CNSR PARAMCD PARAM;
run;

海外の最新動向：FDA Project OptimusとCDISC IG

1. FDA Project Optimus と MIDD

FDAは、癌治療薬の開発において最適な用量選択を推進するため、2021年に「Project Optimus」を立ち上げました。このプロジェクトは、特に癌臨床試験において「最大耐用量（MTD）＝最適用量」という従来の考え方を見直し、早期から薬物動態学（PK）、薬力学（PD）、曝露反応関係に基づいた最適な生物学的用量（Biologically Optimized Dose）を特定することを求めています。

主な要求事項: * 用量反応関係の評価: 癌治療薬の承認申請では、有効性と安全性の両面から用量反応関係を包括的に評価したデータ提出が必須となります。 * MIDDの活用: 母集団PK/PDモデル、シミュレーション、臨床試験のデザイン最適化など、MIDDアプローチを積極的に活用し、用量選択の科学的根拠を強化するよう推奨しています。 * 複数用量の検討: 早期臨床試験（フェーズ1/2）で複数の用量を検討し、その結果を用いて用量反応モデルを構築することが期待されています。

出典: FDA Project Optimus

2. CDISC ADaM Implementation Guide (ADaM IG) の更新

CDISCは、FDAのProject Optimusのような規制動向に対応するため、ADaM標準の更新を続けています。最新のADaM IGでは、曝露反応解析や用量選択をサポートするための新しい変数や推奨事項が追加されています。

特に、PK/PD解析やモデル解析を目的としたADaM BDSの設計に関するガイダンスが強化されており、曝露量（例：AUC、Cmax）やPKパラメータを効果的に格納するための変数定義、それらと臨床アウトカム（例：腫瘍反応、有害事象）を結合する方法が詳細に説明されています。

出典: CDISC ADaM Implementation Guide

日本での対応ポイント：PMDA要件と国内実務

PMDA（医薬品医療機器総合機構）も、医薬品開発におけるMIDDの重要性を認識しており、特に新たな薬物開発ガイドラインや審査の場面で、その活用を期待しています。FDAのProject Optimusのような具体的なイニシアチブはまだありませんが、PMDAは科学的妥当性に基づいた用量選択の根拠を常に求めています。

1. PMDAへの提出資料におけるADaMの活用

PMDAは、承認申請資料の一部としてCDISC標準のデータ提出を推奨しています。特に、用量反応関係を裏付ける解析データは、ADaM形式で提出することが求められます。これには、以下の点が重要です。 * 明確なトレーサビリティ: SDTMからADaMへのデータの変換プロセスがDefine-XML（データセットのメタデータや変数の詳細を記述するXMLファイル）で明確に記述され、PMDA審査官が容易にデータの由来や導出ロジックを理解できるようにすること。 * PK/PD解析データの充実: PK/PD解析で用いた生データおよび解析結果（例：個体推定値、モデル診断プロット）を、ADaM BDSとして整理し、Define-XMLでその構造と内容を詳細に説明すること。 * 用量選択の科学的根拠: 申請用量がどのように選択されたのか、その根拠となる用量反応モデルやシミュレーション結果を、ADaMデータと紐付けて説明することが不可欠です。

2. FDA要件とPMDA要件の違いと補足

FDAはProject Optimusを通じて、明確に「最適用量選択」を早期開発段階から強く推奨し、その根拠をMIDDで示すことを求めています。一方、PMDAは同様の趣旨を「医薬品開発と承認審査の質の向上」という観点から、各案件の特性に応じて求めている傾向があります。両者の根底にある思想は共通していますが、PMDAはより個別の相談を通じて詳細なガイダンスを提供することが多く、早期段階での薬事相談が非常に有効です。

日本の製薬企業やCROの実務担当者は、FDAの最新ガイダンスを参考にしつつ、PMDAとの薬事戦略会議や対面助言を積極的に活用し、提出データセットの準備を進めることが賢明です。

3. 国内実務へのアドバイス

  • 早期からのADaM設計: PK/PD解析担当者と統計解析担当者が密接に連携し、臨床試験計画の段階からMIDD解析に必要なADaM変数を特定し、設計に盛り込むことが重要です。
  • 標準化された導出ロジック: 累積用量や曝露量カテゴリなど、用量選択に資する派生変数の導出ロジックを標準化し、Define-XMLに詳細に記述することで、規制当局からの質問に迅速に対応できます。
  • ツールとバリデーション: Pinnacle 21などのバリデーションツールを導入し、データ作成の各段階でCDISC準拠性を確認することで、規制当局への提出準備を効率化できます。

よくあるエラーと対処法

1. Pinnacle 21（vaersa）エラー：変数の欠落・不一致

エラー内容: ADaM IGで必須とされている変数が欠落している、または定義されていない変数が存在する、データ型がDefine-XMLと一致しないといったエラー。 対処法: Define-XMLを常に最新のADaM仕様書（ADaM IG）と照合し、すべての必須変数（例：USUBJID, PARAMCD, AVAL, AVISITN）が正しく定義され、値が入力されているかを確認してください。また、SASプログラムで変数を導出する際に、Define-XMLで定義されたデータ型（文字型/数値型、長さ）と一致するように注意してください。

2. MIDD解析用データの論理エラー：曝露量とエンドポイントの不整合

エラー内容: PKデータや曝露量と、臨床エンドポイント（例：腫瘍縮小、有害事象）の測定時点が一致しない、または被験者ごとに不整合がある。 対処法: MIDD解析では、曝露と反応の関連性が重要であるため、タイムラインの整合性が極めて重要です。SDTMデータ（EX, CM, PC, VS, RSなど）からADaMデータを導出する際に、各イベントの日付/時間情報を厳密に管理し、必要に応じてintck関数などで期間を計算する際は、基準日を明確にしてください。複数回の測定がある場合は、適切な集約方法（例：平均、最大値、特定の時点の値）を選択し、Define-XMLでそのロジックを明記します。

3. Define-XMLとの整合性不足

エラー内容: ADaMデータセットは作成されているものの、Define-XMLにデータセットや変数のメタデータが正しく記述されていない、またはSASデータセットの構造とDefine-XMLの記述が一致しない。 対処法: SASプログラムでADaMデータセットを作成したら、必ずそのデータセットの構造（変数名、ラベル、型、長さ）がDefine-XMLの記述と完全に一致するかを確認するプロセスを導入してください。自動生成ツールやマクロの活用も有効です。また、導出変数の派生ロジック（Derivation Method）をDefine-XMLに詳細に記述することで、審査官が理解しやすくなります。

よくある質問（FAQ）

Q1: MIDD解析用のADaMはどこまで詳細に作るべきですか？

A1: MIDD解析の目的と、規制当局に示したい科学的根拠のレベルによります。基本的には、用量選択の根拠となるPK/PDパラメータ、曝露量、そしてそれらと関連する臨床エンドポイント（有効性、安全性）を統合し、解析しやすい形にまとめる必要があります。個体レベルのPKパラメータ（例：AUC、Cmax）だけでなく、それらから導出されるカテゴリー変数（例：曝露量グループ）や、時間イベント解析に必要な期間変数なども重要です。解析で直接使用する変数のみを厳選し、Define-XMLでその導出ロジックを詳細に記述することが求められます。

Q2: 癌臨床試験における用量選択をサポートするADaM変数には、具体的に何が必要ですか？

A2: 主に以下の変数が重要です。 * 曝露量関連変数: SDTMのEXドメインから導出される累積投与量（CUMDOSE）、PKドメインから得られるAUCやCmaxなどのPKパラメータ（BDSのAVAL）、それらをカテゴリ化した曝露量グループ（DOSECATEG）。 * 用量情報: 投与された薬剤の用量（DOSE）、投与計画（DOSEPPL）、治療アーム（TRT01Aなど）。 * 有効性エンドポイント: 腫瘍縮小率（AVALとして）、無増悪生存期間（PFS）や全生存期間（OS）などの時間イベントデータ（ADTTEのATPT, CNSR）。 * 安全性エンドポイント: 有害事象の重症度、頻度、発現期間など。 これらの変数をADSL、ADaM BDS、ADTTEなどに適切に配置し、連携させることが重要です。

Q3: 海外規制当局（FDA、EMA）とPMDAでMIDDデータ提出に違いはありますか？

A3: 提出データの「内容」と「形式」においては、大きな違いはありません。CDISC標準（SDTM, ADaM, Define-XML）はグローバルで共通認識されています。しかし、「強調されるポイント」や「審査の焦点」には若干の違いが見られます。 FDAはProject Optimusを通じて、癌臨床試験における用量選択の科学的根拠としてMIDDの積極的な活用を強く推奨し、提出資料への反映をより明確に求めています。PMDAも同様の目的を持っていますが、個別の案件での相談を通じて、より詳細なガイドラインや期待値が示されることが多いです。そのため、日本の製薬企業はPMDAとの薬事相談を早期から行い、必要に応じて海外規制当局の動向も参考にしながら、最適なデータ提出戦略を構築することが重要です。
