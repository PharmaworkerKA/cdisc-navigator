PMDA承認審査と臨床データ提出とは？

医薬品や医療機器の承認を得るためには、その安全性と有効性を示す膨大な臨床試験データをPMDA（医薬品医療機器総合機構）に提出する必要があります。このデータ提出は現在、eCTD（申請電子データパッケージ）と呼ばれる電子形式で行うことが義務付けられています。eCTDは、申請資料全体を構造化されたXML形式で提出する国際的な標準であり、臨床試験データもその一部として含まれます。

CDISCデータ標準の基礎

臨床データは、そのまま提出するのではなく、CDISCが定める国際的な標準形式に変換して提出することが求められます。これは、審査官がデータを効率的に理解し、審査の透明性と比較可能性を高めるためです。主なCDISC標準には以下のものがあります。

  • SDTM（Submission Data Tabulation Model）: 個々の被験者データ（例：人口統計、バイタルサイン、有害事象、併用薬など）を生データに近い形で構造化するためのモデルです。ドメインと呼ばれる複数のデータセットで構成されます。
  • ADaM（Analysis Data Model）: 統計解析に直接使用できるよう、SDTMデータを解析目的に合わせて加工・派生させた解析用データセットのモデルです。解析結果の再現性確保に貢献します。
  • Define-XML: SDTMやADaMといったデータセットの構造、変数、コードリスト、導出ロジックなどを定義するメタデータファイルです。データセットの「取扱説明書」のような役割を果たし、審査官がデータを理解するための必須資料です。
  • CDISC CT（Controlled Terminology）: SDTMやADaMで使用される変数値（例：性別、人種、有害事象の程度など）に、標準化された用語リストを提供します。これにより、データの一貫性と比較可能性が保たれます。

これらの標準に準拠したデータ提出は、PMDA承認審査を円滑に進める上で不可欠な要素です。適切なデータ標準の適用は、データ品質の向上だけでなく、審査期間の短縮にも寄与すると考えられます。

PMDAへの臨床データ提出：実務でのステップとSASコード例

ここでは、CDISC準拠の臨床データを作成し、PMDAへ提出するまでの実務的なステップを、SASコード例を交えて解説します。

ステップ1：SDTMデータセットの準備

SDTMデータセットは、生データ（EDCシステムなどから出力されたデータ）をCDISC標準のドメイン構造に合わせてマッピング（変換）することから始まります。各ドメイン（DM, AE, CMなど）には必須変数があり、それらを正確に設定することが重要です。

SDTMマッピングの具体例（DMドメイン）

```sas data sdtm.DM; set raw_data.DEMOGRAPHICS; / 生データソース /

/ 必須変数のマッピング / USUBJID = catx('-', 'SITE', put(SITEID, z4.), put(SUBJID, z4.)); / 試験固有被験者識別子 / SUBJID = put(SUBJID, z4.); / 施設内被験者識別子 / STUDYID = 'ABC001'; / 試験識別子 / DOMAIN = 'DM'; / ドメインコード / RFSTDTC = put(RFSTDTC, yymmdd10.); / 被験者初回投与日時 / RFENDTC = put(RFENDTC, yymmdd10.); / 被験者最終投与日時 / BRTHDTC = put(BRTHDTC, yymmdd10.); / 誕生日 /

/ その他の変数のマッピング / SEX = SEX_RAW; / 性別 / AGE = int((today() - BRTHDTC)/365.25); / 年齢を計算 / RACE = RACE_RAW; / 人種 / ETHNIC = ETHNIC_RAW; / 民族 / ARM = put(TREATMENT_GRP, $TREATGRPFMT.); / 割付アーム / ACTARM = ARM; / 実際の治療アーム / SITEID = put(SITEID, z4.); / 施設識別子 /

/ ラベルとフォーマットの設定 / label USUBJID = 'Unique Subject Identifier' STUDYID = 'Study Identifier' DOMAIN = 'Domain Abbreviation' RFSTDTC = 'Date/Time of First Study Treatment' RFENDTC = 'Date/Time of Last Study Treatment' BRTHDTC = 'Date of Birth' SEX = 'Sex' AGE = 'Age' RACE = 'Race' ETHNIC = 'Ethnicity' ARM = 'Planned Arm' ACTARM = 'Actual Arm';

format RFSTDTC RFENDTC BRTHDTC yymmdd10.; run;

proc contents data=sdtm.DM; run;
