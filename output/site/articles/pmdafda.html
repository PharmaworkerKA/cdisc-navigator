結論から言うと

医薬品の承認審査を成功させるためには、PMDAとFDA双方で臨床データ提出の要件を満たすことが不可欠です。CDISC標準に準拠したSDTM/ADaMデータセットとDefine-XMLをeCTD形式で提出し、各規制当局のStudy Data Technical Conformance Guideを遵守した品質の高いデータを用意することが、迅速な承認への鍵となります。

承認審査と臨床データ提出とは？

新薬が患者さんのもとに届くためには、安全性と有効性を科学的に証明し、各国の規制当局（日本ではPMDA、米国ではFDA、欧州ではEMAなど）の承認審査を通過する必要があります。この審査において、臨床試験で得られた膨大なデータは、その評価の根幹となります。従来は紙媒体や非標準化された形式での提出も許されていましたが、グローバルな開発が進むにつれて、効率的かつ一貫したデータレビューを可能にするための標準化が強く求められるようになりました。

そこで登場したのが、CDISC（Clinical Data Interchange Standards Consortium）データ標準です。CDISCは、臨床データ提出の際に使用するデータの構造や形式を国際的に標準化することで、規制当局によるレビューの効率化、データの再利用性の向上、そして国際共同治験における協力体制の強化を目指しています。

主要なCDISC標準には、生データに近い構造で臨床試験データを標準化するSDTM（Study Data Tabulation Model）、統計解析に用いる解析データを標準化するADaM（Analysis Data Model）、そしてデータセットのメタデータ（定義情報）をXML形式で記述するDefine-XMLがあります。これらの標準に則って作成されたデータは、eCTD（electronic Common Technical Document：電子共通技術文書）という形式で規制当局に提出されます。eCTDは、医薬品の申請に必要な文書を電子的に構造化するための国際的な標準フォーマットです。

PMDAやFDAは、このCDISC標準に準拠した電子的臨床データ提出を義務付けており、そのための具体的な要件を「Study Data Technical Conformance Guide（技術的適合性ガイド）」などの文書で示しています。これにより、提出データの品質と一貫性が保たれ、より迅速かつ的確な承認審査が可能になります。

実務での使い方・実装手順

CDISC準拠の臨床データ提出は、SASプログラマーにとって日々の業務の核となる部分です。ここでは、SDTMとADaMデータセットの作成、Define-XMLの準備、そしてeCTDへの組込み手順について、具体的なSASコード例を交えながら解説します。

1. SDTMデータセットの作成

SDTM（Study Data Tabulation Model）は、個々の被験者データ（Observation Level Data）を標準化するためのモデルです。生データ（CRFデータやラボデータ）をSDTMドメイン（例: DM, AE, CM, VS, LBなど）にマッピングし、標準変数（Variable）と制御語彙（Controlled Terminology：CT）を用いて構造化します。

SASコード例: DM（Demographics）ドメインの作成

```sas / SDTM DMドメインの作成例 / data sdtm.dm; set raw.patient_info; / 元の症例登録情報データセット /

/ USUBJID (Unique Subject Identifier): 施設と被験者番号を組み合わせてユニークなIDを作成 / USUBJID = catx('-', 'STUDYID001', SITEID, SUBJID); STUDYID = 'STUDY001'; / 試験識別子 / DOMAIN = 'DM'; / ドメインコード / SUBJID = put(SUBJID, 8.); / 被験者識別子を文字列化（必要に応じて） /

/ RFSTDTC (Date/Time of First Study Treatment): 最初の治験薬投与開始日 / RFSTDTC = put(date_of_first_dose, yymmdd10.); / RFENDTC (Date/Time of Last Study Treatment): 最後の治験薬投与終了日 / RFENDTC = put(date_of_last_dose, yymmdd10.);

/ BRTHDTC (Date of Birth): 生年月日 / BRTHDTC = put(birth_date, yymmdd10.); / AGE (Age (Years)): 年齢を計算 / AGE = int((today() - birth_date) / 365.25);

/ SEX (Sex): 性別をCDISC CTにマッピング / SEX = case; when gender = 'M' then 'MALE'; when gender = 'F' then 'FEMALE'; else 'UNKNOWN'; end;

/ RACE (Race): 人種をマッピング / RACE = raw_race; / ETHNIC (Ethnicity): 民族をマッピング / ETHNIC = raw_ethnicity;

/ DTHFL (Subject Death Flag): 死亡フラグ / DTHFL = if death_flag = 'Y' then 'Y'; / DTHDTC (Date/Time of Death): 死亡日時 / DTHDTC = put(death_date, yymmdd10.);

/ 変数ラベルの付与 (Define-XMLの生成にも使用される重要なメタデータ) / label USUBJID = 'Unique Subject Identifier' STUDYID = 'Study Identifier' DOMAIN = 'Domain Abbreviation' SUBJID = 'Subject Identifier' RFSTDTC = 'Date/Time of First Study Treatment' RFENDTC = 'Date/Time of Last Study Treatment' BRTHDTC = 'Date of Birth' AGE = 'Age (Years)' SEX = 'Sex' RACE = 'Race' ETHNIC = 'Ethnicity' DTHFL = 'Subject Death Flag' DTHDTC = 'Date/Time of Death' ; run;
